## Avant de commencer

Ce manuel est un outil de travail, pas un article de blog. Il a été conçu pour être utilisé avec votre CV ouvert à côté. Chaque section contient des éléments à copier, cocher, ou adapter directement.

**Ce que ce manuel fait**
- Vous aide à passer les filtres ATS et les détecteurs IA
- Vous donne des formulations prêtes à l'emploi
- Vous fournit des prompts testés pour améliorer votre CV avec l'IA
- Vous apprend à parler de vos compétences IA de façon crédible

**Ce que ce manuel ne fait pas**
- Garantir un entretien (le fond compte autant que la forme)
- Remplacer une vraie expérience (si le fond est vide, la forme ne sauvera rien)
- Rester valide éternellement (les détecteurs évoluent, revérifiez dans 6 mois)

> ⚠️ **Avertissement légal** : Les statistiques citées proviennent de sources publiques et d'études sectorielles. Les outils mentionnés sont des suggestions basées sur leur adoption par le marché en janvier 2026. L'auteur n'a aucun lien commercial avec ces outils. Ce document ne constitue pas un conseil juridique ou professionnel.

Avant d'ouvrir votre CV, comprenez le terrain. Ces chiffres expliquent pourquoi ce manuel existe.

| Statistique | Source |
|---|---|
| **75%** des CV sont filtrés par ATS avant lecture humaine | Jobscan, 2025 |
| **62%** des CV contiennent du texte généré par IA | ResumeBuilder, 2025 |
| **48%** des recruteurs utilisent des détecteurs IA dans leur processus | SHRM, 2025 |
| **20%** de callbacks en moins pour les CV flaggés "AI-generated" | NonAI Resumes, 2025 |

Ce n'est pas une question de style. C'est une question de **survie dans le pipeline de recrutement**. Votre CV doit passer trois filtres : l'ATS (mots-clés), le détecteur IA (authenticité), et l'humain (pertinence). Ce manuel traite du deuxième filtre, le plus récent et le moins compris.

## Section 1 : Matrice de Décision

Première question : devez-vous même mentionner l'IA sur votre CV ? La réponse dépend de votre secteur et du poste visé.

| Catégorie | Secteurs / Postes | Recommandation |
|---|---|---|
| **METTRE EN AVANT** | Tech, Data, Produit, Ops, Marketing Digital, **Consulting** | L'IA est attendue. Soyez spécifique : outils, résultats, contexte. C'est un différenciateur positif. |
| **MENTIONNER AVEC PRUDENCE** | Création de contenu, Journalisme, Juridique, RH traditionnelle | L'IA peut être perçue comme "triche" ou manque d'authenticité. Mentionnez-la comme outil d'assistance, pas comme compétence centrale. |
| **ÉVITER / INVISIBLE** | Concours académiques, Sécurité sensible, Postes où l'authenticité est critique (Littérature, Haute Joaillerie, Psychologie clinique) | Ne mentionnez pas l'IA. Utilisez-la en coulisses pour préparer, mais ne la mettez pas sur le CV. |

> **Note pour profil Consulting** : Vous êtes dans la catégorie "Mettre en avant". Ce manuel est fait pour vous.

## Section 2 : Diagnostic Express

Cochez chaque erreur présente dans votre CV. Chaque case cochée = une correction obligatoire.

- [ ] **L'Observateur Vague** : "Suivi des tendances IA", "Exploré le potentiel de l'IA"
- [ ] **L'Accumulateur d'Outils** : Liste d'outils sans contexte ("Utilisé Claude, GPT-5, Zapier...")
- [ ] **Le Généraliste Gonflé** : Langage d'équipe de 30 personnes pour un side project
- [ ] **Le Vrai Travail Sous-vendu** : Bon projet décrit de façon trop générique
- [ ] **La Contribution Générique** : "Collaboré sur l'intégration LLM" sans détails
- [ ] **Le Mixeur de Buzzwords** : Jargon empilé ("pipeline RAG multi-agents zero-shot")
- [ ] **L'Appropriateur Excessif** : S'attribuer le travail d'une équipe entière
- [ ] **La Victoire Déconnectée** : Résultat impressionnant sans expliquer le rôle de l'IA
- [ ] **L'Illusion Prompt-comme-Produit** : Un prompt manuel présenté comme un logiciel
- [ ] **L'Usage Banalisé** : "Utilisation quotidienne de ChatGPT" (tout le monde en 2026)

**Score** : 0-2 = Bon départ | 3-5 = Réécriture partielle | 6+ = Réécriture complète

## Section 3 : Clean-Up Room

Ctrl+F chaque mot de la Blacklist dans votre CV. S'il apparaît, remplacez-le immédiatement.

### BLACKLIST : Mots à bannir

Ces mots sont surreprésentés dans les textes ChatGPT. Les détecteurs les flaggent.

| Mot interdit | Pourquoi c'est un problème |
|---|---|
| Exploiter / Leverage | Verbe favori de ChatGPT. Ne dit rien de concret. |
| Synergie | Jargon corporate détecté comme IA. |
| Transformatif | Adjectif vague, surexploité par les LLM. |
| Crucial / Critique | Inflation de l'importance, pattern IA classique. |
| Paysage (sens figuré) | "Le paysage de l'IA" = traduction littérale de "landscape". Marqueur IA. |
| Plonger (delve) | Marqueur ChatGPT majeur en anglais, traduit en français. |
| Est un témoignage de | Traduction de "is a testament to". Tournure artificielle. |
| De pointe | "Cutting-edge" traduit. Adjectif creux. |
| Optimiser | Verbe vague sans action concrète. |
| Rationaliser | "Streamline" traduit. Trop corporate. |
| Innovant | Auto-proclamation sans preuve. |
| Stratégique | Flou intentionnel, red flag pour recruteurs. |
| Holistique | Buzzword corporate typique. |
| Passionné | Opinion, pas un fait. Impossible à vérifier. |
| Proactif | Générique et surexploité. |

### GREENLIST : Verbes d'action humains

Ces verbes impliquent friction, responsabilité et travail réel. Ils passent les détecteurs.

| Verbe | Exemple d'utilisation |
|---|---|
| Construit | Construit un pipeline de classification avec Claude pour 2 000 tickets/mois |

| Négocié | Négocié l'accès API avec la DSI pour le pilote, budget 2K€/mois validé |
| Cassé | Cassé puis reconstruit le prompt après 3 échecs en production |
| Documenté | Documenté les 12 cas d'échec identifiés pour l'équipe support |
| Mesuré | Mesuré la précision à 89% sur 500 cas annotés manuellement |
| Refactoré | Refactoré les prompts pour réduire la consommation de tokens de 35% |
| Arbitré | Arbitré contre le déploiement IA sur flux paiement après analyse risque |
| Sécurisé | Sécurisé le pipeline avec filtre PII avant envoi vers l'API externe |

## Section 5 : Prompt Library

Ces prompts sont conçus selon les standards d'ingénierie : rôle défini, exemples fournis, structure de sortie spécifiée. **Ce ne sont pas des questions posées à l'IA. Ce sont des algorithmes textuels.** Copiez l'intégralité du bloc dans Claude ou ChatGPT.

### Prompt #1 : Le Brutal Editor

*Usage : Transformer une formulation floue en bullet factuel.*

```
Agis comme un recruteur technique senior et sceptique,
spécialisé dans les profils tech et produit.

Je vais te donner une section de mon CV.
Ta tâche : Réécris chaque point pour éliminer tout
"fluff" (jargon vide, adjectifs subjectifs comme
"passionné" ou "stratégique").

Contraintes de format :
1. Structure obligatoire : Verbe d'action fort +
   Outil technique spécifique + Résultat mesurable
2. Si une métrique manque, insère [insérer % ou temps]
3. Ton sec, factuel et professionnel
4. Maximum 2 lignes par point

Exemple de ce que je veux :
Mauvais : "J'ai utilisé l'IA pour améliorer la
productivité de l'équipe marketing."
Bon : "Développé script Python + API GPT-5 pour
générer balises SEO, réduisant temps rédaction de 20%."

Voici mes points à réécrire :
[COLLER VOS BULLET POINTS ICI]
```

### Prompt #2 : Le Détecteur de Fluff

*Usage : Identifier ce qui est du remplissage et obtenir un score de densité.*

```
Analyse le texte suivant en te mettant dans la peau
d'un Responsable RH fatigué de lire des CV générés
par l'IA.

Procède étape par étape :
1. IDENTIFICATION : Relève toutes les phrases qui sont
   des "Opinions" (ex: "excellente capacité") au lieu
   de "Faits" vérifiables.
2. DÉTECTION BLACKLIST : Surligne les mots qui sonnent
   trop IA : paysage, plonger, témoignage, transformatif,
   crucial, exploiter, synergie, holistique, proactif.
3. SCORE DE DENSITÉ : Note sur 100.
   (100 = 100% faits vérifiables, 0 = 100% blabla)

Sois impitoyable. Je veux savoir ce qui sonne faux.

Texte à analyser :
[COLLER VOTRE RÉSUMÉ OU EXPÉRIENCE ICI]
```

### Prompt #3 : Le Générateur de Maturité Opérationnelle

*Usage : Ajouter des bullets "senior" sur sécurité, coûts, et gestion des risques.*

```
Je veux ajouter une dimension "Maturité Opérationnelle"
à mon CV pour rassurer un employeur sur ma gestion
des risques liés à l'IA.

Contexte : Je suis [VOTRE MÉTIER].
Ta tâche : Propose 5 bullet points concrets que je
pourrais adapter, démontrant ma compétence sur :
1. SÉCURITÉ & CONFIDENTIALITÉ
   (anonymisation, RGPD, IA locale, filtres PII)
2. CONTRÔLE DES COÛTS
   (optimisation tokens, choix du bon modèle, caching)
3. GESTION DES HALLUCINATIONS
   (validation humaine, tests, fallbacks, garde-fous)

Format : Phrases prêtes à l'emploi, commençant par un
verbe au passé (Mis en place, Sécurisé, Audité, etc.)
Génère les propositions maintenant.
```

## Section 6 : Mur de la Maturité

Ces phrases montrent que vous comprenez les enjeux réels : sécurité, coûts, risques, gouvernance. Copiez et adaptez avec vos vrais contextes.

### Sécurité & Conformité

- Mis en place un filtre PII (données personnelles) avant envoi vers l'API OpenAI, assurant conformité RGPD.
- Déployé instance locale de Mistral-7B pour traitement documents confidentiels sans connexion externe.
- Documenté les flux de données IA pour audit SOC 2 : inputs, outputs, rétention, contrôle d'accès.
- Créé processus de validation humaine obligatoire pour tout output IA destiné aux clients.
- Négocié clause de confidentialité spécifique avec le fournisseur API pour les données sensibles.

### Coûts & Performance

- Refactoré prompts système pour réduire consommation tokens de 35% à performance égale.
- Implémenté cache de réponses pour requêtes récurrentes, réduction appels API de 60%.
- Migré tâches simples de GPT-5 vers GPT-5o-mini : coût divisé par 10, précision maintenue à 94%.
- Mis en place dashboard de suivi des coûts API par équipe et par cas d'usage.
- Négocié tarif volume avec OpenAI après avoir démontré ROI positif sur 3 mois de pilote.

### Gestion des Risques & Robustesse

- Ajouté logique de fallback vers réponse humaine après 2 échecs consécutifs du modèle.
- Construit framework d'évaluation interne : 500 cas de test, seuil déploiement fixé à 85% précision.
- Documenté les 12 cas d'échec identifiés en production et créé garde-fous spécifiques pour chacun.
- Arbitré contre déploiement IA sur flux paiement après analyse risque/bénéfice : trop de faux positifs.
- Mis en place alerting automatique quand le taux d'erreur dépasse 5% sur une fenêtre de 24h.

## Section 7 : Checklist Anti-Détecteur

Avant d'envoyer votre CV, passez par ces 5 étapes. Dans l'ordre.

### Étape 1 : Le Test Automatique
- [ ] Aller sur un détecteur Tier 1 (voir Section 8)
- [ ] Coller l'intégralité du texte de votre CV
- [ ] Score > 50% "AI-generated" : passer aux étapes suivantes
- [ ] Score < 20% : OK, vérification finale seulement

### Étape 2 : La Chasse aux Mots Interdits
- [ ] Ctrl+F chaque mot de la Blacklist (Section 3)
- [ ] Remplacer chaque occurrence par un équivalent Greenlist
- [ ] Vérifier : aucun exploiter, synergie, transformatif, crucial, stratégique

### Étape 3 : La Variation de Rythme
- [ ] Relire à voix haute : toutes les phrases ont la même longueur ?
- [ ] Ajouter au moins 2 phrases courtes (moins de 8 mots)
- [ ] Ajouter au moins 1 phrase longue avec subordonnée
- [ ] Éviter 3 phrases consécutives commençant par le même mot

### Étape 4 : L'Injection de Friction
- [ ] Ajouter au moins 1 mention d'échec ou difficulté surmontée
- [ ] Ajouter au moins 1 arbitrage ou décision difficile
- [ ] Remplacer au moins 2 verbes "lisses" par verbes "rugueux"

> **Verbes lisses** : géré, coordonné, supervisé, assuré  
> **Verbes rugueux** : débogué, cassé, tranché, arbitré

### Étape 5 : Le Re-Test Final
- [ ] Repasser le texte modifié dans le détecteur
- [ ] Score cible : moins de 30% "AI-generated"
- [ ] Si échec : réécrire manuellement les sections encore flaggées

## Glossaire

| Terme | Définition |
|---|---|
| **API** | Application Programming Interface. Interface permettant à un logiciel de communiquer avec un autre. Dans le contexte IA, c'est ce qui permet d'envoyer des requêtes à GPT-5 ou Claude depuis votre code. |
| **ATS** | Applicant Tracking System. Logiciel utilisé par les recruteurs pour filtrer les CV par mots-clés avant lecture humaine. 75% des CV sont rejetés à cette étape. |
| **Chunking** | Découpage d'un document en segments plus petits pour le traitement par un LLM. Un mauvais chunking = mauvais résultats RAG. |
| **Fallback** | Mécanisme de secours. Exemple : si l'IA échoue 2 fois, le système redirige vers un humain. |
| **Function Calling** | Capacité d'un LLM à appeler des fonctions externes (recherche, calcul, API) pendant sa réponse. |
| **Hallucination** | Quand un LLM génère une information fausse mais présentée avec confiance. Problème majeur à gérer en production. |
| **LLM** | Large Language Model. Modèle de langage de grande taille comme GPT-5, Claude, ou Mistral. |
| **PII** | Personally Identifiable Information. Données personnelles (nom, email, téléphone) à filtrer avant envoi vers une API externe pour conformité RGPD. |
| **Pipeline** | Chaîne de traitement automatisée. Exemple : document → chunking → embedding → stockage → requête → réponse. |
| **Prompt** | Instruction textuelle envoyée à un LLM. L'art du prompt engineering consiste à formuler des instructions qui produisent des résultats précis. |
| **RAG** | Retrieval-Augmented Generation. Architecture qui combine recherche dans une base de documents + génération par LLM. Permet de répondre sur des données spécifiques. |
| **Reranker** | Modèle qui réordonne les résultats d'une recherche par pertinence. Améliore la précision d'un système RAG. |
| **Retry Logic** | Logique de réessai automatique en cas d'erreur (timeout, rate limit). Essentiel pour la robustesse en production. |
| **SOC 2** | Standard de conformité pour la sécurité des données. Audit requis pour les entreprises traitant des données sensibles. |
| **Tokens** | Unités de texte utilisées par les LLM (environ 4 caractères = 1 token). Les API facturent au token. Optimiser les tokens = réduire les coûts. |
